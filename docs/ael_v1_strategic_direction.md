# AEL v1.0 Strategic Direction

**Date:** 2026-03-20
**Status:** Active Direction
**Drafted by:** Andrew Li

---

## 1. Objective

Transition AEL from:

> AI-assisted tool

to:

> AI-driven engineering system

---

## 2. Strategic Shift

### From

- human workflow optimization
- toolchain compatibility
- interface usability

### To

- AI capability expansion
- autonomous execution
- closed-loop intelligence

---

## 3. Non-Goals

We will not prioritize:

- IDE-like interfaces
- GUI complexity
- traditional abstraction layers
- human-centric workflow improvements

---

## 4. Core Focus Areas

### 4.1 Capability Model

Define hardware in terms of:

- signal
- timing
- behavior

### 4.2 Structured Observation

All results must be:

- machine-readable
- comparable
- analyzable

### 4.3 Experimentation

Enable AI to:

- run multiple trials
- vary parameters
- compare results

### 4.4 Autonomous Loop

Support:

- generate -> run -> observe -> fix -> repeat

### 4.5 Memory System

Store:

- failures
- fixes
- patterns

---

## 5. Success Criteria

AEL v1.0 succeeds if:

- AI can complete full hardware validation autonomously
- human input is reduced to goal specification
- the system produces explainable results with evidence

---

## 6. Philosophy

> Do not teach users a new workflow.
> Build a system so powerful that users abandon the old one.
