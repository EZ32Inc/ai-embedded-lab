# User-Facing Response Checklist v0.1

Use this checklist before finalizing a response to user questions about boards, instruments, test creation, or test execution.

## Pre-Response Checklist
1. **Capability**: Did I start by explaining what AEL can do in this scenario? [ ]
2. **Instrument Role**: Did I explain the role of the instrument or board? [ ]
3. **Execution Plan**: Did I present a step-by-step plan before acting? [ ]
4. **Constraint Statement**: Did I state realistic limitations (e.g., capture capability)? [ ]
5. **Request to Proceed**: Did I end with a clear offer to continue? [ ]
6. **No Command Dump**: Did I avoid dumping raw shell commands or scripts? [ ]
7. **No Implicit Assumptions**: Did I avoid treating repo config as user reality? [ ]

## Readiness Evaluation
Does the answer follow:
**Capability + Role + Plan + Constraint + Proceed**? [ ]

---

## When to Use This Checklist
- General "how to" inquiries about boards/instruments.
- Requests to create or run new tests.
- Requests to use AEL for new hardware.
