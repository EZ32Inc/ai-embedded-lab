# Generated Example First Execution Sprint v0.1

## Purpose

This note defines the first post-governance execution sprint for generated
examples.

## Sprint Goal

Remove one concrete execution blocker, not several at once.

## Recommended Sprint Scope

Pick one of these:

1. Provision one UART runtime path and run one generated UART example.
2. Complete one explicit ADC external-input contract for one generated ADC
   example.

## What This Sprint Should Not Do

- broad multi-example runtime execution
- broad ESP32 generated-example runtime promotion
- USB expansion
- new vendor-family expansion
- another governance-only cycle

## Success Criterion

At the end of the sprint, one example should move one step forward:

- from blocked-by-setup to runtime-attemptable, or
- from blocked-by-contract to contract-complete
